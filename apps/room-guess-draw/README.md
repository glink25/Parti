# 你画我猜房间设计说明

本文档记录 `room-guess-draw` 当前版本的产品定位、玩法规则、状态模型和实现边界，作为后续迭代的产品设计基线。若文档与代码不一致，以当前版本代码为准，并应在修改代码时同步更新本文档。

## 1. 产品定位

“你画我猜”是一个 2–8 人实时联机绘画聚会房间。所有玩家使用自己的设备加入同一 Parti 房间，通过共享画布、私密题目、文字猜测和同步揭晓完成游戏。

产品提供两种玩法：

- **经典模式**：玩家依次担任画手，其他玩家实时看画并抢答，通过答题速度累计积分。
- **接力模式**：所有玩家同时进行“看词作画”和“看画猜词”，内容在玩家之间轮转，最终按画册逐步揭晓传播过程。

房间不允许旁观者。产品不依赖摄像头、麦克风、剪贴板、外部网络或持久化存储。

## 2. 通用设计原则

### 2.1 入场与准备

- 房间支持 2–8 名玩家，房主也是参与者。
- 大厅、经典模式结算页和接力画册总览页允许玩家准备或取消准备。
- 至少两名在线玩家且所有在线玩家都准备后，系统自动开始游戏。
- 只有房主可以在可准备阶段切换经典模式与接力模式。
- 开局后新加入的玩家在经典模式可进入当前回合猜题；接力模式中等待下一局加入，避免破坏已经建立的画册轮转关系。

### 2.2 题库

内置题库分为八个分类，共包含 240 个题目，每类 30 题。分类配置和题目数据统一位于 `src/word-bank.ts`，游戏 Worker 只负责抽取和运行逻辑：

- `daily`：日常生活，常见物品、设施和场景。
- `animals`：动物世界，外形和习性容易通过绘画表达的动物。
- `imagination`：脑洞趣味，动作、状态、成语和想象组合。
- `food`：美食天地，主食、甜点、水果和饮品。
- `sports`：运动时刻，球类、田径、户外和竞技动作。
- `places`：地点场景，建筑、交通、公共空间和旅行地点。
- `jobs`：职业人物，外形与工作动作鲜明的常见职业。
- `culture`：文化节庆，传统习俗、手工艺和适合绘画的成语故事。

每个题目包含稳定 ID、答案和两条文字提示。系统按分类记录已使用题目，在候选题耗尽前尽量不重复；耗尽后重置该分类记录，并尽量避开刚使用过的题目。

### 2.3 画布能力与约束

- 画手可使用画笔或橡皮擦、6 种固定颜色和 3 档笔刷大小，并可清空整张画布。
- 坐标使用 `0–1` 归一化值，使不同尺寸设备看到一致画面。
- 单张画布最多保留 300 条笔画、累计最多 24,000 个点，单条笔画最多 420 个点。
- 绘制中的当前整笔路径每 300 毫秒至多提交一次，松手时立即提交最终版本；每名玩家每秒最多提交 35 次画笔 action。
- Worker 校验工具、颜色、粗细、坐标、笔画 ID、阶段和操作者权限，不信任客户端 payload。
- 经典模式画布通过每 300 毫秒一次的完整笔画公开快照同步；接力模式的画布彼此私密，在揭晓前不得进入公开状态。

## 3. 经典模式

### 3.1 整局流程

1. 所有在线玩家准备，系统随机生成本局画手顺序。
2. 当前画手在 20 秒内选择题目分类；超时则系统随机选择分类。
3. Worker 从分类中选择题目，通过私人事件只告诉画手答案。
4. 画手在 90 秒内作画，其他玩家实时输入答案或聊天。
5. 所有仍在线的猜题玩家答对，或 90 秒倒计时结束时，本回合结束。
6. 结算页公开答案和本轮得分，停留 5 秒后自动切换下一位画手。
7. 本局画手顺序中的在线玩家都完成一次作画后结束，展示最终积分排名。

断线玩家会在轮转时被跳过。画手断线后本回合仍保留其位置，房间恢复逻辑会在必要时重新选题或推进到下一位。

### 3.2 提示

作画开始后系统按固定时间公开提示：

| 作画经过时间 | 提示内容 |
| --- | --- |
| 30 秒 | 答案字数 |
| 55 秒 | 题目的第一条语义提示 |
| 75 秒 | 题目的第二条语义提示 |

提示作为公开状态和系统消息展示，不修改答案判定。

### 3.3 猜词与聊天

- 画手不能猜自己的题；本回合猜题名单中的其他玩家可以提交最多 40 个字符的文本。
- 文本会去除首尾空白、合并连续空格，并在比较答案时忽略大小写、空格和常见中英文标点。
- 每名玩家两次消息之间至少间隔 650 毫秒。
- 首次完全匹配答案时判定猜对；重复提交答案不重复得分。
- 未完全匹配、但文本包含完整答案时不广播，避免在聊天中直接泄露答案。
- 猜对后仍可继续参与聊天，但不会再次得分。
- 聊天、系统提示、正确通知和答案揭晓共享消息流，最多保留最近 60 条。

### 3.4 积分

猜题者按答对时的剩余时间获得积分：

| 剩余时间 | 猜题者得分 |
| --- | ---: |
| 大于 60 秒 | 20 |
| 大于 35 秒 | 15 |
| 大于 15 秒 | 10 |
| 15 秒及以下 | 5 |

每有一名玩家首次猜对，画手额外获得 2 分。最终排名按累计积分从高到低显示。

## 4. 接力模式

### 4.1 产品目的

接力模式不以竞速和积分为核心，而是让每条词语在“画作”和“猜测”之间不断转换，最终形成容易跑偏、适合共同观看和互动的画册。

所有开局玩家组成固定轮转环。每名玩家既拥有一条自己发起的画册，也会在每一步收到另一条画册的最新内容。

### 4.2 接力流程

1. **选择题目**：每名玩家在 20 秒内选择分类，系统分别私密分配不同题目；超时者随机选分类和题目。
2. **初始作画**：每名玩家根据自己的私密题目同时作画，限时 60 秒，也可提前提交。
3. **看画猜词**：画作按固定轮转关系传给下一名玩家；接收者只看画，不看原题，在 25 秒内提交猜测。
4. **按猜测重画**：猜测继续传递，下一名玩家只看到上一条文字猜测，并在 60 秒内重新作画。
5. **继续交替**：系统重复“看画猜词”和“按猜测重画”，每一步都向同一方向传递，确保玩家不会处理自己刚提交的内容。
6. **最终猜测**：当画册即将走完所有其他玩家时，最后一名接收者只看上一幅画并提交最终文本；提交或超时后画册立即完成，不再产生下一次绘画任务。两人局由第二名玩家完成这一步后直接进入揭晓。
7. **逐步揭晓**：所有画册完成后，由房主依次公开原题、每幅画和每次猜测。
8. **画册总览**：全部揭晓后保留所有完整画册，玩家可发送表情反应并准备下一局。

绘画或猜测阶段全员提交后立即推进；到达时限时，未提交的画作按当前内容保存，未提交的猜测记为“未猜出”。

### 4.3 私密边界

- 原始题目、当前绘画提示、未揭晓画作、当前猜测和画册归属只存在于 Worker 模块内存。
- Worker 使用定向事件向每名玩家发送当前任务，玩家不能从公开快照读取其他画册内容。
- 公开状态只包含阶段、参与者、步数、截止时间、已提交玩家和揭晓进度。
- 进入揭晓阶段后，画册内容才按房主操作逐项写入公开状态。
- 房主可以重播当前揭晓画作；所有玩家可以发送 `😂`、`🤯`、`🎨`、`👏`、`❓` 五种瞬时表情反应。

接力模式不计算胜负或积分；界面中的侧栏在游戏期间用于展示接力完成进度。

## 5. 联机状态与秘密数据

### 5.1 公开状态

公开快照包含以下信息：

- 当前模式、阶段、房主和玩家在线/准备状态。
- 经典模式的积分、画手顺序、当前画手、分类、倒计时、提示、猜对名单、消息、已完成笔画和当前活动笔画的累计点列。
- 题库分类、各分类已使用题目 ID 和最近题目 ID。
- 接力模式的固定参与者、当前步骤、已提交名单、阶段截止时间和公开揭晓数据。

公开状态不得在进行中包含经典模式答案、接力私密任务或未揭晓画册内容。

### 5.2 Worker 隐藏状态

Worker 模块内存保存：

- 经典模式当前题目和正在绘制但尚未结束的笔画。
- 接力画册完整链路、玩家当前分配、猜测、私密画布、活动笔画和本局已预留题目。
- 聊天与画笔 action 的频率限制记录。

当前 `schema` 值仅标识本版本状态结构，不要求实现跨版本迁移或兼容层。

## 6. 加入、离开与恢复

- 大厅或经典结算阶段离开的玩家会从玩家和积分数据中删除。
- 游戏进行中离开的玩家保留席位并标记为离线，重连时恢复原身份和名字。
- 经典模式画手重连时，如果 Worker 仍持有答案，会重新发送私人题目。
- 接力模式参与者重连时，如果 Worker 内存仍在，会重新发送其当前私密任务和画布。
- 房主运行环境从持久化快照恢复时，模块内秘密无法恢复：
  - 接力局直接作废并回到大厅，所有玩家重新准备。
  - 经典局当前题作废；画手仍在线则重新选词，否则推进到下一位。

## 7. 房间协议

### 7.1 Actions

| Action | 主要权限 | 作用 |
| --- | --- | --- |
| `selectMode` | 房主 | 在可准备阶段切换 `classic` 或 `relay` |
| `toggleReady` | 在线玩家 | 准备、取消准备或准备下一局 |
| `chooseCategory` | 当前画手/接力参与者 | 选择题目分类 |
| `submitGuess` | 猜题者/接力参与者 | 提交经典猜测或接力文字猜测 |
| `submitStroke` | 当前画手 | 使用 `{ canvasRevision, taskId?, updateSeq, complete, stroke }` 低频提交完整当前笔画 |
| `clearCanvas` | 当前画手 | 清空自己的当前画布 |
| `relaySubmitDrawing` | 接力画手 | 提前提交当前画作 |
| `advanceReveal` | 房主 | 推进接力画册揭晓 |
| `replayRevealDrawing` | 房主 | 重播当前揭晓的画作 |
| `sendReaction` | 所有玩家 | 在揭晓或总览阶段发送表情反应 |

### 7.2 私人和瞬时事件

| Event | 接收者 | 作用 |
| --- | --- | --- |
| `guess:word` | 经典模式画手 | 发送当前秘密答案 |
| `guess:notice` | 单个猜题者 | 限流、答案保护或得分反馈 |
| `relay:task` | 单个接力参与者 | 发送其当前私密题目、画作或重绘提示 |
| `relay:submitted` | 单个接力参与者 | 确认当前内容已经提交 |
| `relay:replay` | 所有玩家 | 重播当前揭晓画作 |
| `relay:reaction` | 所有玩家 | 展示瞬时表情反应 |

## 8. 视觉与交互原则

- 整体为轻快的纸张与手绘工具风格，使用橙色主操作、浅色画布、柔和蓝绿状态色。
- 画布是首要内容，聊天输入和积分/进度侧栏保持紧凑，桌面与移动端都尽量在单屏完成主要操作。
- 使用归一化画布和响应式重绘适配不同屏幕比例。
- 当前私密题目只出现在对应玩家画布角落，其他玩家看到相同位置时不得出现答案。
- 倒计时剩余 10 秒时进入紧急状态，同时使用颜色和脉冲动画提示。
- 已准备、已提交、离线、猜对和当前玩家状态同时通过文字或符号表达，不只依赖颜色。
- 接力揭晓强调横向时间线，让原题、画作、猜测和创作者关系清晰可追溯。

## 9. 代码与打包结构

| 文件 | 职责 |
| --- | --- |
| `index.html`、`src/main.tsx` | Vite 入口与 React 应用装载 |
| `src/App.tsx`、`src/DrawingBoard.tsx` | 阶段组件、交互界面和高频 Canvas 绘制 |
| `src/useParti.ts`、`src/types.ts` | 集中的 Parti 订阅、私密事件和协议类型 |
| `src/worker.ts`、`src/game-logic.ts` | 权威状态机、隐藏数据与可测试的纯游戏逻辑 |
| `public/parti.room.json` | 房间元信息、人数、权限、入口和 action 声明 |
| `vite.config.ts` | 构建 React UI，并用 esbuild 合并单文件 Worker |
| `package.json` | 构建、Room 开发、类型检查和纯逻辑测试脚本 |

该房间使用 `filesystem` Package。Vite 构建 React 资源，esbuild 将 Worker 及其纯逻辑依赖合并成运行时要求的单一 `worker.js`。

画布使用 `react-konva` 的矢量 `Line` 渲染。画师设备本地即时绘制，每 300 毫秒向 Worker 提交一次完整的当前 Line，松手时立即提交最终版本。经典模式仅通过公开快照展示 `strokes + activeStroke`，不使用瞬时绘画事件或点批次；某次更新丢失时，下一份完整 Line 会自然覆盖恢复。`updateSeq` 用于拒绝重复和过期更新，`canvasRevision` 用于拒绝跨回合数据。接力画布在揭晓前仍只存在于 Worker 隐藏内存和对应玩家的私密任务中。

页面以 `360×640` 为最小布局基线，固定在一个 `100dvh` 视口中。画布、工具栏、玩家状态、聊天记录和输入框始终同时存在；历史聊天和揭晓时间线只在各自容器内部滚动。准备、选词、结算和揭晓操作由权威阶段驱动的模态层展示。

## 10. 验证方式

常用命令：

```bash
pnpm room:dev room-guess-draw
pnpm --filter @parti/room-guess-draw typecheck
pnpm --filter @parti/room-guess-draw test
PARTI_ROOM_BUILD_OUT_DIR=/tmp/room-guess-draw-package pnpm --filter @parti/room-guess-draw build:room
```

自动检查应关注完整 Package 是否输出到 Harness 提供的目录，以及 manifest、UI 和 Worker 入口是否齐全。

浏览器中的 Canvas 手感、触控绘画、多设备同步、倒计时节奏、接力隐私、断线重连和响应式布局必须由真人玩家在目标设备上验证。

## 11. 当前明确不做的内容

- 旁观模式、机器人玩家或单人练习。
- 语音、视频、图片上传或摄像头辅助绘画。
- 自定义题库、玩家手动输入原始题目或用户生成内容审核。
- 撤销/重做、图层、形状、文字、填充等高级绘图工具。
- 经典模式跳过题目、画手主动结束回合或房主中途强制推进。
- 接力模式积分、胜负、投票评选或自动生成分享图片。
- 历史战绩、跨房间排行榜、长期保存或导出画作。
- 跨版本房间数据迁移和兼容处理。
