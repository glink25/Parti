# 元素训练场（Magicka）

`room-magicka` 是 Parti 中用于验证元素组合法术、状态反应和多人战斗同步的 1–4 人合作 Demo。玩家在固定的伪横版 X/Y 战斗带中点击移动，通过依次选择最多四个元素生成法术，与持续刷新的怪物战斗。

当前版本使用基础碰撞盒和 Canvas 图形，不包含正式美术、动画、音频、关卡、掉落或持久成长。它的主要目标是提供一套可扩展、可测试的玩法框架。

## 当前功能

- 七种元素：岩、火、冰、生命、雷、水、盾。
- 有序元素队列，最多四槽，允许重复元素。
- 精确配方优先、通用组合兜底的法术解析。
- 投射物、扇形、射线、指定地点、持续区域、全局环境和自身施法七种载体。
- 潮湿、燃烧、寒冷、冻结、感电和护盾状态。
- 击退、击飞、减速和眩晕控制效果。
- 近战、远程和水系混合三类怪物，包含攻击前摇、冷却和敌方投射物。
- 本地即时伤害反馈、浮动伤害数字、血条、受击闪烁和状态标签。
- 倒地、生命法术救援、全员倒地结算和房主重开。
- 房主迁移后从最近一次共享快照继续运行。

## 操作方式

### 移动

元素队列为空时，点击战斗区域设置移动目标。角色沿直线移动到目标点，不进行自动寻路。

### 元素与施法

- 点击底部元素按钮选择元素；桌面端也可使用数字键 `1–7`。
- `Backspace` 撤销最后一个元素，`Escape` 清空队列。
- 元素队列非空时，按住战斗区域瞄准法术。
- 非射线法术在松手时提交，经过法术的 `castMs` 后自动触发。
- 普通法术吟唱期间可以准备下一组元素，但不能提交另一法术。
- 射线法术需要持续按住：先吟唱，完成后开始喷射；手指移动会平滑改变射线方向，松手立即结束。
- 潮湿状态下直接加入雷元素会使自身感电并打断施法；先加入盾元素可以组成接地法术。

## 技术架构

```text
输入与 Canvas UI
    │
    ▼
MagickaScene（客户端模拟与表现）
    ├── spells.ts：元素队列 → SpellPlan
    ├── combat.ts：几何、运动、控制、怪物上限
    └── contracts.ts：共享类型和协议
    │
    ▼
Parti action / event
    │
    ▼
Worker（房间阶段、快照存储、事件转发）
```

### 文件职责

| 路径 | 职责 |
| --- | --- |
| `src/scenes/MagickaScene.ts` | 输入、施法状态机、本地玩家模拟、房主世界模拟、碰撞结算和 Canvas 绘制 |
| `src/game/contracts.ts` | 元素、法术、状态、实体、快照和网络消息类型 |
| `src/game/spells.ts` | 基础法术、精确配方、通用组合解析和状态反应规则 |
| `src/game/combat.ts` | 射线/扇形/圆形几何、角度追踪、控制运动和人数怪物上限 |
| `src/worker/index.ts` | 房间生命周期、玩家快照、房主世界快照和事件广播 |
| `public/parti.room.json` | 房间元数据、权限和 action 接口声明 |

客户端使用 LittleJS 提供的 Canvas 和游戏循环，但战斗实体、法术碰撞和 UI 均由当前项目直接管理。Worker 通过 `@parti/worker-sdk` 接入 Parti 房间。

## 法术系统

### 解析流程

```text
有序元素队列
  → 查找完全匹配的 SpellRecipe
  → 未命中时使用首元素的基础载体
  → 后续元素强化范围、距离和伤害
  → StaffModifier（预留）
  → SpellPlan
```

元素顺序有意义。例如 `火 + 生命` 命中“生命烈焰”配方，而 `生命 + 火` 会进入通用生命法术组合。

`SpellPlan` 包含：

- `delivery`：法术载体。
- `castMs`：吟唱时间。
- `range`、`radius`、`coneAngle`：几何参数。
- `projectileSpeed`：投射物速度。
- `pierce`：额外穿透目标数量。
- `tickMs`：射线伤害间隔。
- `recoveryMs`：施法后摇。
- `effects`：伤害、治疗、状态、控制或环境效果。

### 载体行为

| 载体 | 行为 |
| --- | --- |
| `projectile` | 生成本地投射物，逐帧移动，碰撞目标或到期后销毁 |
| `cone` | 以角色和瞄准方向构造扇区，一次命中范围内所有目标 |
| `beam` | 从角色向瞄准方向延伸至场地边界；默认被首个怪物阻挡，按 Tick 持续伤害 |
| `targeted` | 吟唱结束后在记录的目标点进行一次范围结算 |
| `area` | 在目标点结算并可生成持续环境区域 |
| `global` | 产生全场环境效果，例如降雨 |
| `self` | 直接作用于施法者，不创建瞄准连线 |

### 当前精确配方

| 组合 | 法术 |
| --- | --- |
| 火 + 生命 | 生命烈焰射线 |
| 火 + 岩 + 岩 + 火 | 天降陨石 |
| 火 + 水 + 水 + 火 | 全场降雨 |
| 水 + 冰 | 冻结水流 |
| 火 + 水 | 蒸汽云 |
| 岩 + 火 | 熔岩弹 |
| 水 + 生命 | 生命之泉 |
| 岩 + 冰 | 碎冰岩弹 |
| 水 + 雷 | 导电水链 |
| 火 + 火 | 烈焰喷射 |
| 盾 + 岩/火/冰/雷/水 | 对应防御护盾 |

新增特殊配方只需向 `RECIPES` 添加数据。新增载体则需要同时扩展 `Delivery`、场景中的执行逻辑、预览/正式表现和几何测试。

## 施法状态机

```text
Idle
 ├── 空队列点击 → MoveTarget
 └── 有队列按下 → AimState
                         │
        ┌────────────────┴────────────────┐
        │ 非射线松手                       │ 射线持续按住
        ▼                                  ▼
   PendingCast                    吟唱完成 → ActiveBeam
        │                                  │
   triggersAt 到达                         └── 松手
        ▼                                      │
   执行法术 → Recovery ◄───────────────────────┘
```

- `AimState` 保存按下指针、目标和方向。
- `PendingCast` 固化松手时的起点和目标，避免后续移动改变法术目标。
- `ActiveBeam` 保存当前角度、目标角度和下一次伤害时间。
- 眩晕、击飞和潮湿感电会清除瞄准、待触发法术和活动射线。
- 后摇期间可以选择元素，但不能提交新法术。

## 状态、反应与控制

### 状态

`StatusInstance` 记录状态种类、强度、层数、来源和结束时间。重复状态会刷新持续时间并累积层数，上限为五层。

当前反应：

| 目标状态 | 新伤害 | 结果 |
| --- | --- | --- |
| 潮湿 | 火 | 伤害提高并显示“蒸发” |
| 潮湿 | 雷 | 雷伤提高；雷法术同时造成眩晕 |
| 冻结 | 岩 | 物理伤害提高并显示“碎裂” |

降雨会周期性使玩家和怪物潮湿。潮湿玩家选择雷元素时会自身感电，除非当前队列已包含盾。

### 控制运动

`ControlState` 同时用于本地玩家和怪物：

- `knockback`：向作用方向增加 X/Y 速度，随后受阻力衰减。
- `knockup`：增加视觉高度 `z` 和垂直速度 `vz`，落地前无法行动。
- `slow`：在持续时间内乘算移动速度。
- `stun`：在持续时间内禁止移动、攻击和施法。

怪物的控制运动由房主模拟；玩家的控制运动由各自客户端模拟。

## 怪物系统

| 类型 | 行为 |
| --- | --- |
| `chaser` | 追击最近玩家，进入距离后进行带前摇的近战攻击 |
| `shooter` | 与目标保持距离，前摇后发射远程投射物 |
| `water-fiend` | 近距离使用击退攻击，远距离发射附加潮湿/减速的水弹 |

怪物攻击使用 `EnemyAttackState` 保存攻击类型、目标、开始和生效时间。前摇期间会绘制进度环，攻击被眩晕或击飞时取消。

怪物生成由房主负责。仅当存活数量低于硬上限时生成新怪物：

| 已连接玩家 | 怪物上限 |
| --- | --- |
| 1 | 8 |
| 2 | 12 |
| 3 | 16 |
| 4 | 20 |

生成间隔会随本局持续时间缩短，但不会突破人数上限。

## 多人同步模型

当前版本有意采用“客户端自主结算、房主汇总共享世界”的轻量模型，不考虑反作弊。

### 所有权划分

| 数据 | 负责方 |
| --- | --- |
| 本地玩家位置、生命、状态、控制和倒地 | 该玩家客户端 |
| 玩家法术、瞄准、投射物和命中检测 | 施法者客户端 |
| 怪物生成、AI、控制运动和攻击 | 房主客户端 |
| 敌方投射物运动 | 房主客户端 |
| 玩家是否被怪物攻击命中 | 被攻击玩家客户端 |
| 怪物最终生命与共享环境 | 房主客户端汇总 |
| 房间阶段与最新快照 | Worker |

### 怪物伤害数据流

```text
施法者本地碰撞
  → 立即扣除本地怪物镜像生命并显示反馈
  → reportEnemyDamage(eventId, damage, statuses, controls)
  → Worker 广播 magicka:enemy-damage
  → 房主去重并更新共享怪物
  → 房主下一次 syncWorld 发布最终世界快照
```

本地命中会记录预期怪物生命；收到房主确认后的快照前，旧快照不会覆盖这部分预测伤害。怪物生命归零时立即从本地集合删除。正常房主运行期间不会反向合并旧 Worker 世界，避免死亡怪物被上一帧快照重新加入；只有首次接任房主时才加载共享世界。

### 快照与高频事件

- `publishPlayer`：玩家约每 100ms 发布自身公开状态。
- `syncWorld`：房主约每 100ms 发布怪物、敌方投射物和环境。
- `reportEnemyDamage`：广播怪物伤害、状态和控制结果。
- `reportPlayerEffect`：向目标玩家广播非伤害效果。
- `reportEnvironment`：请求房主加入天气或区域效果。
- `reportProjectileHit`：通知房主销毁已命中的敌方投射物。

`sequence` 和 `worldSequence` 用于丢弃旧快照，`eventId` 用于客户端事件去重。Worker 不验证伤害数值、位置、距离或轨迹。

### 房主迁移

Worker 始终在快照中发布当前 `hostId`。客户端首次发现自己成为房主时，从最近快照接管怪物、投射物和环境，随后只以自己的世界模拟为准。非房主持续以 Worker 快照刷新共享世界镜像。

## 房间生命周期

```text
lobby → 所有已连接玩家准备 → running
running → 所有已连接玩家倒地 → gameover
gameover → 房主点击重开 → lobby
```

倒地玩家停止移动和施法。生命法术将其生命恢复到阈值后可完成救援。

## 扩展指南

### 新增法术配方

1. 在 `spells.ts` 的 `RECIPES` 中添加有序元素组合。
2. 尽量复用现有 `Delivery` 和 `SpellEffect` 原语。
3. 为顺序、重复元素和伤害/状态反应补充测试。

### 新增元素或效果原语

1. 更新 `contracts.ts` 中的联合类型和 `ELEMENTS`。
2. 在 `spells.ts` 中定义基础载体与组合行为。
3. 在 `MagickaScene` 中实现效果结算和表现。
4. 检查 Worker 的 payload 过滤是否需要识别新字段。

### 新增怪物

1. 扩展 `EnemyKind` 和必要的攻击类型。
2. 在房主模拟中定义生成属性、移动策略、攻击距离、前摇和冷却。
3. 为特殊投射物或控制增加共享类型。
4. 保持生成逻辑受 `enemyCap()` 硬上限约束。

## 已知限制

- 当前同步模型信任客户端，不具备反作弊能力。
- 玩家位置和怪物快照只做直接刷新，没有完整的远端插值或延迟补偿。
- 友军不会承受直接法术伤害，但可接收状态、治疗和控制效果。
- 法术和敌方投射物使用基础几何碰撞，不包含复杂障碍物或地形遮挡。
- 环境系统目前只有降雨和蒸汽区域。
- `StaffModifier` 类型已经预留，但 Demo 尚未提供可切换法杖。
- UI 以横屏为目标，没有独立竖屏布局。

## 开发与验证

从仓库根目录执行：

```sh
# 启动房间开发模式
pnpm room:dev room-magicka

# 规则与几何测试
pnpm --filter @parti/room-magicka test

# TypeScript 检查
pnpm --filter @parti/room-magicka typecheck
```

独立验证房间构建：

```sh
PARTI_ROOM_BUILD_OUT_DIR=/tmp/parti-room-magicka-build \
  pnpm --filter @parti/room-magicka build:room
```

提交新机制前至少应覆盖：

- 法术配方顺序和通用回退。
- 新载体的几何边界与命中顺序。
- 新状态或控制的叠加、结束和打断规则。
- 多人伤害事件去重及旧快照回滚。
- 怪物攻击前摇、冷却、投射物销毁和人数上限。
