# 谁是卧底房间设计说明

本文档记录 `room-undercover` 当前版本的产品设计、游戏规则和实现边界。若文档与代码不一致，以代码为准，并应在修改代码时同步更新本文档。

## 1. 产品定位

本房间负责选词、隐藏发牌、记录出局和判断胜方，不负责语音、计时、应用内投票或计分。

- **联机模式**：每名玩家使用自己的设备加入房间，系统私密发送词牌。
- **牌盒模式**：在一台设备上生成序号牌盒，玩家轮流查看自己的牌；该模式不修改联机房间状态。
- 支持 3–12 名收牌玩家，不允许旁观者加入。

## 2. 游戏规则

### 2.1 身份与描述

- `civilian`（好人）：获得相同的好人词。
- `undercover`（卧底）：获得另一个相近词；多名卧底的词相同。
- `blank`（白板）：没有词。

玩家轮流描述。描述不得包含自己词语中的字，否则视为自爆并立即出局；白板没有该限制。玩家可以在不说出词中字的前提下自由描述，不要求描述符合自己的词。每轮线下投票处死一名玩家，然后继续下一轮。

应用不会监听或判断描述，也不实现应用内投票。玩家被投票或自爆出局后，在界面自行确认出局。

### 2.2 胜负顺序

每次确认一名玩家出局后，按以下顺序结算：

1. 若存活卧底为 0：白板仍存活则白板胜，否则好人胜。
2. 若仍有卧底，且存活好人数小于或等于存活卧底数，则卧底胜。
3. 其他情况继续游戏。

白板不计入好人人数。没有白板的普通对局在所有卧底出局后由好人获胜。

### 2.3 白板出现规则

白板是独立发牌选项，不是词语模式。未启用时沿用普通好人/卧底分配；启用后：

| 收牌人数 | 白板出现概率 | 命中后的角色组成 |
| --- | ---: | --- |
| 3 | 25% | 2 好人、1 白板；第一次出局即结算 |
| 4 | 50% | 2 好人、1 卧底、1 白板 |
| 5 | 75% | 3 好人、1 卧底、1 白板 |
| 6–9 | 100% | 1 卧底、1 白板，其余好人 |
| 10–12 | 100% | 2 卧底、1 白板，其余好人 |

3–5 人每次发牌独立随机，不做保底。实际是否命中只保存在隐藏牌中，游戏进行期间不公开，结算时才揭晓。

白板看到的提示为：

> **你是白板**
>
> 没有词，也不会因说中词里的字而自爆。
>
> 请保持镇定，假装一切尽在掌握。

### 2.4 普通卧底人数

未出现白板时按以下规则发卧底：3–5 人 1 名、6–9 人 2 名、10–12 人 3 名。内置词对每轮有 50% 概率交换好人与卧底方向；自定义词方向固定。

## 3. 发牌与交互

### 3.1 联机模式

发牌台提供两种词语来源：

- `classic`：从所选内置分类抽取词对，房主与其他玩家都参与收牌。
- `custom`：房主填写两个非空、不同且不超过 20 个字符的词，房主主持且不收牌。

“加入白板”是两种来源共用的独立开关，不会自行排除房主。经典词库启用白板后，房主仍然收牌。私人事件 `undercover:card` 只发送 `{ round, word }`，不发送角色。

收牌玩家点击“我出局了”并确认后提交 `round:eliminateSelf`。身份在结算前保持隐藏。结算公开胜方、两个词以及本局是否实际存在卧底和白板。

中途加入者等待下一轮；重连会重发仍在内存中的私人牌。房主运行环境从快照恢复时隐藏牌无法恢复，因此重置为等待重新发牌。

### 3.2 牌盒模式

牌盒模式使用经典内置词库，支持 3–12 人、分类选择和独立白板开关。首次点击序号查看私人牌；再次点击可重看或确认出局。白板牌使用与联机模式相同的提示。所有角色、随机结果与结算均保存在当前页面内存中，刷新后丢失。

## 4. 状态与协议

公开 `RoomState` 包含：阶段、房主和玩家列表、词语来源、是否启用白板、分类、轮次、收牌与出局列表，以及完成后的胜方和揭晓信息。进行中不公开角色、词语或低人数局是否实际出现白板。

隐藏 Worker 数据使用：

```ts
type Role = 'civilian' | 'undercover' | 'blank';
type Winner = Role;
type DealResult = {
  cards: Record<string, { role: Role; word: string }>;
  words: { civilian: string; undercover: string };
  hadBlank: boolean;
  hadUndercover: boolean;
};
```

Actions：

| Action | 权限 | 作用 |
| --- | --- | --- |
| `settings:setMode` | 房主 | 设置 `classic` 或 `custom` |
| `settings:setIncludeBlank` | 房主 | 开关白板选项 |
| `settings:setCategories` | 房主 | 设置至少一个有效分类 |
| `round:deal` | 房主 | 校验设置、洗牌并开启新一轮 |
| `round:eliminateSelf` | 本轮收牌玩家 | 确认投票或自爆出局并触发结算 |

私人事件：`undercover:card`，payload 为 `{ round: number; word: string }`。

## 5. 实现边界

- 纯逻辑位于 `src/game-logic.ts`，由联机 Worker 与牌盒模式共同使用。
- 只为确定性纯逻辑编写测试，不为 UI、动画、网络或完整游戏流程引入测试框架。
- 游戏体验与实际联机流程由人工试玩验证，不使用浏览器自动化作为权威验证。
- 只实现当前版本数据结构，不设计跨版本房间数据迁移、兼容层或旧数据回退。
